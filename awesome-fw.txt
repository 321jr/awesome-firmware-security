FIXME: Summer reading list


Awesome Firmware
https://github.com/FIXME

Awesome Firmware is a curated list of firmware resources, with a focus on security and development, and an emphasis on UEFI/ACPI and related platform firmware. This initial list is a single list of terms and tools, alphabetized. Future versions will be grouped into multiple subsections.

==========

ACPI
http://uefi.org/acpi/
ACPI is a platform firmware technology, originally intended to replace Plug and Play, MP, and Advanced Power Management. The UEFI Forum is the current owner of this technology. ACPI is used on both BIOS and ACPI systems, and is used by Intel/AMD and now [optionally] ARM. Acpidump, CHIPSEC, and FWTS are the main ACPI test tools.

ACPICA
https://acpica.org/
The ACPI Component Architecture Project (ACPICA) provides a collection of cross-platform ACPI tools, such as acpidump.

acpidump
https://acpica.org/
Acpidump is one of a handful of tools by ACPICA, a set of cross-platform OS-present tools to diagnose ACPI tables.

AMD PSP
https://en.wikipedia.org/wiki/AMD_Platform_Security_Processor
The AMD PSP (Platform Security Processor) is a security processor on AMD systems, which runs firmware applications such as fTPM.

Android Verified Boot
https://source.android.com/security/verifiedboot/
Verified Boot is a firmware security technology from Google, that helps secure the boot process. Verified Boot is used on both Android (Android Verified Boot, aka AVB) and on ChromeOS. Verified Boot is roughly equivalent to UEFI Secure Boot.

Apple T2
https://www.apple.com/imac-pro/
The Apple T2 is a security processor on some newer Apple systems, which runs provides additional firmware security than older Apple systems.

ARC
https://en.wikipedia.org/wiki/Advanced_Computing_Environment
ARC (Advanced Computing Environment) is a platform firmware technology used by early Windows NT non-Intel systems. The design of ARC was influential to the design of UEFI: firmware images on a hard disk partition, pointed to by variables.

BadBIOS
https://en.wikipedia.org/wiki/BadBIOS
BadBIOS is the alleged firmware malware reported by Dragos.

BIOS
https://en.wikipedia.org/wiki/BIOS
BIOS is a platform firmware technology initially used on the Intel-based IBM PC. It is an 8086 Real Mode technology. Intel has said that they will end-of-life BIOS-based platform firmware by 2020, replacing it with UEFI. Intel and a few IBVs have closed-source BIOS implementations, SeaBIOS is the main open source implementation. BIOS used to be the main firmware techology on Windows PCs, until Windows started requiring UEFI. The Intel/ARM-based -- but not ARM-based -- Google Chrome PCs use coreboot with a BIOS payload.

Baseboard Management Controller (BMC)
https://en.wikipedia.org/wiki/Intelligent_Platform_Management_Interface#Baseboard_management_controller
A BMC is an interface to manage -- including update -- server firmware. OpenBMC is the main open source BMC implementation.

Beyond BIOS: Developing with the Unified Extensible Firmware Interface, Third Edition
https://www.degruyter.com/view/product/484468
This is a book on UEFI by Intel and other UEFI Forum members. Originally published by Intel Press.

CHIPSEC
https://github.com/chipsec/chipsec
CHIPSEC is a security tool created by Intel, to test the security posture of Intel hardware/firmware. It is currently the only tool that can check for multiple public firmware security vulnerabilities. There are currently only a few active UEFI parsers, CHIPSEC, UEFITool, and UEFI Firmware Parser.

ChromeOS Verified Boot
https://www.chromium.org/chromium-os/chromiumos-design-docs/verified-boot
Verified Boot is a firmware security technology from Google, that helps secure the boot process. Verified Boot is used on both Android (Android Verified Boot, aka AVB) and on ChromeOS. Verified Boot is roughly equivalent to UEFI Secure Boot.

coreboot
https://coreboot.org/
coreboot is a platform firmware technology, originally called LinuxBIOS. It loads payloads such as SeaBIOS, UEFI, among others. Coreboot -- and U-Boot -- are widely used in embedded systems. Coreboot is used by Google on ChromeOS systems, using coreboot Verified Boot for additional security.

DarkReading article on firmware security
https://www.darkreading.com/iot/5-tips-for-protecting-firmware-from-attacks/d/d-id/1325604
This article, which has input from the Intel CHIPSEC team, gives basic high-level guidance for firmware security. Start with this, before digging into the NIST documents.

DASH
http://www.dmtf.org/standards/dash/
DMTF DASH is an out-of-band firmware management technology for desktops, similar to IPMI, Intel AMT, and DMTF Redfish. DMTF SMASH is related to DASH.

DMA
https://en.wikipedia.org/wiki/Direct_memory_access
DMA is a system memory feature that is attackable by Rogue Hardware such as PCIleech. Modern operaging systems have to protect against DMA attacks, some OSes now have configuration features to help protect against DMA attacks.

eficheck
https://apple.com/
Eficheck is a new UEFI security tool from Apple, available in recent versions of macOS.

Evil Maid Attack
http://theinvisiblethings.blogspot.com/2011/09/anti-evil-maid.html
The Evil Maid attack is perhaps the most well-known firmware attack, where the victim leaves their sstem unattended and an attacker has some period of time with physical access to the system, for them to install firmware-level malware. For example, person leaves their laptop in their hotel room while out for dinner, and the attacker is posing as hotel room service.

Expansion ROM
https://en.wikipedia.org/wiki/Option_ROM
See Option ROM.

FirmWare Test Suite
https://launchpad.net/fwts
FirmWare Test Suite (FWTS) is a collection of firmware tests created by Canonical, the Ubuntu Linux OSV, to help test a system for defects that will cause Ubuntu problems. FWTS is a suite of dozens of tests, for multiple technologies. The UEFI Forum recommends FWTS as the main ACPI test resource. FWTS is a command line tool for Linux, and includes an optional CURSES UI, and an optional FWTS-live live-boot distribution. FWTS is included in LUV.

Firmware Security blog
https://firmwaresecurity.com
This blog is a source of firmware security (and development) news and information. This weblog generates a few short blog posts every few days with the latest news on firmware security and development, focusing mainly on UEFI-centric platform firmware, and focusing a bit more on Linux than other OSes. (DISCLAIMER: One of the awesome-firmware authors is the Firmware Security blogger.]

Firmware-Security List on Twitter
https://twitter.com/JacobTorrey/lists/firmware-security
Jacob Torrey hosts this list on Twitter, which contains many of the core firmware security researchers.

FlashROM
https://flashrom.org/Flashrom
FlashROM is a UNIX-centric online firmware tool that dumps the platform firmware image to a file, for later offline analysis. Flashrom works on Linux/BSD systems, including the WinFlashROM Windows port.

Hacking Team's UEFI malware
https://attack.mitre.org/wiki/Software/S0047
Hacking Team is a company that sells exploits to governments and others. Amongst their offerings is a UEFI-based firmware attack for Windows PCs. The Hacking Team malware is one of the few existing known public UEFI blacklisted by CHIPSEC.

Harnessing the UEFI Shell: Moving the Platform Beyond DOS, Second Edition
https://www.degruyter.com/view/product/484477
This is a book on UEFI by Intel and other UEFI Forum members. Originally published by Intel Press.

Golden Image
https://en.wikipedia.org/wiki/System_image
A golden image is the vendor's original binaries for the firmware. The term is also used for OS images. Most users only use the latter definition. Better vendors provide images and tools to reset used hardware/Grey Market acquisitions to a known state. Before trusting any downloaded binary, such as a golden image, it should be compared to a hash. Most vendors do not provide a hash for their images.

IBV
https://en.wikipedia.org/wiki/BIOS#Vendors_and_products
An Independent BIOS Vendor (IBV) provides an integrated firmware solution to OEMs/ODMs.  With UEFI replacing BIOS, some IBVs now refer to themselves as IFVs, Independent Firmware Vendors.  Some OEMs will outsource their consumer-class device firmware to IBVs, and do their own firmware for their business-class devices.

Intel AMT
https://software.intel.com/en-us/articles/developing-for-intel-active-management-technology-amt
Intel AMT is a platform firmware management technology on Intel systems, based on the Intel ME processor. The fTPM is an example of an AMT application.

Intel Boot Guard (IBG)
https://en.wikipedia.org/wiki/Intel_vPro#Intel_Boot_Guard
Intel Boot Guard is a firmware security technology that helps secure the boot process before UEFI Secure Boot takes place. Once Boot Guard is enabled, it cannot be disabled.

IPMI
https://www.intel.com/content/www/us/en/servers/ipmi/ipmi-home.html
IPMI is a platform firmware management technology on Intel systems. While widely-used, the modern replacement for IPMI is Redfish.

Intel ME
https://en.wikipedia.org/wiki/Intel_Management_Engine
Intel ME is a security processor on Intel systems, which runs AMT firmware applications.

Intel Security Training
https://github.com/advanced-threat-research/firmware-security-training
This contains a snapshot of training from the CHIPSEC team at Intel Advanced Threat Research (ATR) team of Intel Security, before the Intel/MkAfee split. The documents are an EXCELLENT source of information about Intel hardware/firmware security threats, focusing on UEFI and related technologies.

JTAG
https://en.wikipedia.org/wiki/JTAG
JTAG is a hardware interface to chips that allows access to the firmware. It is used by firmware engineers during devlopment, and by Evil Maid attackers when the vendor leaves the JTAG interface exposed in consumer devices.

Linux UEFI Validation (LUV) Linux
https://01.org/linux-uefi-validation
LUV is a Linux distro created by Intel to test UEFI implementation of OEMs. It bundles
CHIPSEC, FWTS, and other firmware tests. LUV is available in binary form as LUV-live, a
live-boot distribution.

Linux Vendor Firmware Service (LVFS)
https://fwupd.org/
The Linux (FIXME) is a firmware update service for Linux OEMs, roughly the equivalent of Windows Update, but for Linux systems.

Low Level PC Attack Papers
http://www.timeglider.com/timeline/5ca2daa6078caaf4
This is the time-based list of hardware/firmware security research.

Management Mode
www.uefi.org/sites/default/files/resources/UEFI_Plugfest_March_2016_AMI.pdf
Management Mode is term used by UEFI to refer to both Intel SMM and ARM TrustZone.

Measured Boot
https://intel.com/
Measured Boot is a firmware security technology from Intel, that uses TCG TPMs to help secure the boot process.

NIST
https://csrc.nist.gov/publications/sp800
NIST is a US Government organization that provides IT guidance for US government systems. NIST provides firmware security guidance in 4 documents: SP 800-147, SP 800-147b, SP 800-155 (draft), and SP 800-193 (draft). This guidance is both for device implementors and for enterprise sysadmins.

OEM
https://en.wikipedia.org/wiki/Original_equipment_manufacturer
An OEM builds and sells original hardware.

ODM
https://en.wikipedia.org/wiki/Original_design_manufacturer
An ODM builds hardware and sells them to OEMs.

Option ROM
https://en.wikipedia.org/wiki/Option_ROM
An Option ROM, aka an Expansion ROM, aka OpROM, aka XROM, is the firmware 'blob' of a PCI/PCIe device. An Option ROM is terminology from BIOS era, when a card would hook the BIOS platform firmware and add additional functionality for the new card. An Option ROM is a driver on the card's flash. On UEFI systems, the driver is a UEFI style driver, instead of a BIOS driver. A card may need multiple drivers, one for each architecture and one for each platform firmware type (BIOS, UEFI, etc).

OSV
https://en.wikipedia.org/wiki/Operating_system
An OSV is an Operating System Vendor, which includes firmware/OS interactions.

Pawn
https://github.com/google/pawn
Google Pawn is a Linux-centric online firmware tool that dumps the platform firmware image to a file, for later offline analysis.

PCIe
https://pcisig.com/
PCIe is the interface for PC boards. PCIe devices include Option ROMs of firmware. The device may have a processor invisible to the system board, it is difficult to fully trust PCIe hardware.

PCILeech
https://github.com/ufrisk/pcileech/
PCILeech is PCI-based rogue hardware used to attack PCI interfaces of systems.

Redfish
http://dmtf.org/standards/redfish/
DMTF Redfish is an out-of-band firmware management technology, replacing IPMI.

Rootkits and Bootkits
https://nostarch.com/rootkits
This is the only book on firmware security at the time, writen by firmware security experts.

Rowhammer
https://en.wikipedia.org/wiki/Row_hammer
Rowhammer is a new form of memory-based security attacks against systems.

RU.EFI
https://github.com/JamesAmiTw/ru-uefi/
RW.EFI is a third-party freeware firmware tool that has multiple features. It works as a MS-DOS or UEFI Shell utility.

RWEverything
http://rweverything.com/
RWEverything (RWE) is a third-party freeware firmware tool that has multiple features. The tools works on Windows. The CHIPSEC tool, if the CHIPSEC Windows kernel driver is not loaded, can use the RWE kernel driver.

SeaBIOS
https://www.seabios.org
SeaBIOS is the main open source implementation of BIOS. Other BIOS implementations are mainly closed-source codebases owned by OEMs and IBVs.

Secure Boot
https://en.wikipedia.org/wiki/Unified_Extensible_Firmware_Interface#Secure_boot
Secure Boot is a term often associated with UEFI Secure Boot, an optional security feature of UEFI that helps secure the boot process. It does not require a TPM. Besides UEFI, other firmware technologies also use the term Secure Boot, sometimes in lower case. The Apple EFI-based Secure Boot implementation is not the same as the Secure Boot technology used by Windows/Linux systems.

SEE
https://en.wikipedia.org/wiki/Trusted_execution_environment
Secure Execution Environment. See TEE.

SMASH
http://dmtf.org/standards/smash/
DMTF SMASH is an out-of-band firmware management technology for servers, similar to IPMI, Intel AMT, and DMTF Redfish. DMTF DASH is related to SMASH.

SMM
https://en.wikipedia.org/wiki/System_Management_Mode
Systems Management Mode (SMM) is a processor mode in Intel and AMD systems, separate from Real and various Protect Modes, that gives full control of the processor. SMM-hosted applications, such as malware, is invisible to the normal Protect Mode-based code.

SPI
https://en.wikipedia.org/wiki/Serial_Peripheral_Interface_Bus
SPI is an interface to accessing the firmware. Used by vendors during development, and used by attackers if left enabled in consumer products.

TEE
https://en.wikipedia.org/wiki/Trusted_execution_environment
Trusted Execution Environment. An example of a hypervisor or other technology that constrains firmware to be more secure. Nearly a synonym for a TEE. ARM TrustZone is an example of a SEE.

ThinkPwn
https://github.com/Cr4sh/ThinkPwn
ThinkPwn is a UEFI malware PoC that originally targets ThinkPad systems. The ThinkPwn malware is one of the few existing known public UEFI blacklisted by CHIPSEC. Thinkpwn.efi is included in FPMurphy's UEFI Utilities, one malware binary amongst other useful tools, be careful if using those tools.

Thunderbolt
https://www.intel.com/content/www/us/en/io/thunderbolt/thunderbolt-technology-general.html
Thunderbolt is a new interface for PCs, using external slots like USB, but based on PCI.

Tianocore
https://tianocore.org/
Tianocore is the home to the UEFI Forum's open source implementation (which vendors use, along with 
their closed-source drivers and value-added code).

TrustZone
https://www.arm.com/products/security-on-arm/trustzone
TrustZone (TZ) is a firmware security technology used on ARM systems, a form of TEE/SEE, called Management Mode by UEFI.

TPM
http://www.trustedcomputinggroup.org/
A TPM is the root-of-trust for many platform firmware implementations, such as Intel/AMD BIOS and UEFI systems. TPM is defined by the Trustworthy Computing Group (TCG). There are discrete TPM chips, as well as fTPM software implementations.

Trusted Boot
https://trustedcomputinggroup.org/trusted-boot/
Trusteded Boot is a firmware security technology from the Trustworthy Computing Group, which uses TPMs to help secure the boot process.

Trustworthy Computing Group (TCG)
http://www.trustedcomputinggroup.org/
The TCG is an industry trade group that controls the TPM and related specifications.

U-Boot
U-Boot is a platform firmare technology. It loads payloads such as SeaBIOS, UEFI, among others. U-Boot -- and coreboot -- are widely used in embedded systems.

UEFI
https://uefi.org
UEFI is a platform firmware technology initially created by Intel, now used by Intel, AMD, ARM, and others, which was initially designed for the Intel Itanium and as a replacement for BIOS. UEFI and BIOS are both implementations of platform firmware technology. UEFI is also EFI. UEFI-based platform firmware technology is often referred to as BIOS, with the older BIOS called Legacy Mode or CSM (Compatibility Support Mode).

UEFI DBX UEFI Secure Boot blacklist file
http://www.uefi.org/revocationlistfile
The UEFI Forum hosts a file that contains the latest UEFI Secure Boot PKI blacklist/expired keys. Check your vendor documentation to see how your system's vendor tools work to obtain and apply this to your system; if the vendor has no tools, ask them to provide them.

UEFI Forum
https://uefi.org/
The UEFI Forum is an industry trade group that controls the UEFI and ACPI specifications, the UEFI SCT tests, and provides the Tianocore open source UEFI implementation.

UEFI Firmware Parser
https://github.com/theopolis/uefi-firmware-parser
UEFI Firmware Parser, which uses the 'uefi-firmware' package name on PyPI, is a tool that parses firmware 'blobs', mainly UEFI ones. There are currently only a few active UEFI parsers, CHIPSEC, UEFITool, and UEFI Firmware Parser.

UEFITool
https://github.com/LongSoft/UEFITool
UEFITool is a GUI program that parses firmware 'blobs', mainly UEFI ones. There are currently only a few active UEFI parsers, CHIPSEC, UEFITool, and UEFI Firmware Parser. In addition to the UEFITool Qt GUI tool, the UEFITool source project also includes a handful of non-GUI command line tools, including UEFIDump. UEFITool has two source trees to be aware of, master and new-engine.

USB
http://www.usb.org
USB is a common port-level interface for PC peripherals. USB devices can be configured to be multiple devices, and rogue USB hardware -- like Hak5's Rubber Ducky -- can trick naieve operating systems.

USB Rubber Ducky
https://hakshop.com/products/usb-rubber-ducky-deluxe
Hak5's Rubber Ducky is an example of rogue USB hardware, which lets the user configure the system to trick naive operating systems into thinking it is any number of devices.

Windows Update
https://en.wikipedia.org/wiki/Windows_Update
Microsoft Windows Update is an update service for Windows OS software, as well as firmware for that system. In practice, most OEMs still use their own tools for firmware updates, instead of Firmware Update.
